Генератор линейных ШК GS1-128

🌙Тёмная

Для сетей розничной торговли. Поддерживает коды с маркировкой и без, разное название/МРЦ для каждой строки.

Формат строки: КОД;Название;МРЦ — название и МРЦ можно не указывать, тогда возьмутся значения по умолчанию сверху (если заданы).
Коды с маркировкой (содержащие 240, 10, 17, 310 и т.д.) обрезаются автоматически — в ШК попадают только (01) и (21).
Ограничения на количество кодов нет, но для стабильной работы браузера рекомендуется до ~500 кодов за один раз.

Ширина — толщина одного штриха (модуля), высота — высота полос в пикселях. Подпись под ШК масштабируется вместе с размером.

Предпросмотр

Что делает приложение

Генерирует линейные штрихкоды GS1-128 (CODE128) по списку кодов: рисует изображение штрихкода с подписью кода, названием товара и МРЦ, и позволяет скачать все картинки одним ZIP-архивом.

Как ввести коды

Вставьте коды в поле «Список кодов» — один код на строку. Строка может быть в одном из трёх форматов:

  1. Только код:
    010400639609867221RU11120526160162049
  2. Код + название + МРЦ (через точку с запятой):
    010400639609867221RU11120526160162049;Chapman Грин OP;285
  3. Код с маркировкой (лишний хвост обрежется автоматически) + название + МРЦ:
    010400639606694721RU11120925345193548 240130400686 10ZIG122610 934000;Chapman Эклипс Нуар Голд;300

Если название или МРЦ не указаны в строке — используются значения по умолчанию сверху формы (если заданы).

Поля формы

  • Название по умолчанию — подставляется, если в строке кода не указано своё название.
  • МРЦ по умолчанию — подставляется, если в строке кода не указана своя цена.
  • Название архива — имя ZIP-файла при скачивании (по умолчанию — «Barcodes»).

Размер штрихкода

  • Ширина штриха (модуль) — толщина одного самого тонкого штриха. Чем больше значение, тем шире весь штрихкод и тем легче он считывается сканером. Допустимо 1–4.
  • Высота (px) — высота полос штрихкода в пикселях (40–200). По стандарту GS1-128 рекомендуемая высота при печати — не менее 32 мм.
  • Подпись под штрихкодом (человекочитаемый код) и строка МРЦ масштабируются автоматически вместе с размером — при увеличении ШК цифры остаются читаемыми.

Стандарт GS1-128

Штрихкоды генерируются строго в формате GS1-128 (он же UCC/EAN-128): сразу после старт-символа вставляется служебный символ FNC1, который отличает GS1-128 от обычного Code 128 и говорит сканеру, что дальше идут данные с идентификаторами применения (AI). Между полями переменной длины при необходимости добавляется разделитель FNC1. Такой код корректно распознаётся кассовыми GS1-сканерами с правильным разделением полей (01), (21) и др.

Ограничение стандарта: не более 48 символов данных в одном штрихкоде (без учёта служебных символов). Если код длиннее — приложение покажет ошибку.

Маркировка (Честный знак)

Если в строке после основного кода GS1 (01…21…) идёт блок маркировки (с AI вида 240, 10, 17, 91, 93 и т.п.), он обрезается автоматически — в штрихкод и подпись попадает только розничный код (01) + (21). Разбор идёт по структуре идентификаторов GS1, а не по случайному совпадению цифр, поэтому «10» или «17» внутри серийного номера не приводят к неверной обрезке.

Совет: если копируете данные прямо из сканера, служебные разделители полей сохраняются и обрезка проходит максимально точно.

Формат подписи (01)/(21)

Под штрихкодом код показывается в читаемом виде — с круглыми скобками вокруг идентификаторов применения GS1, например: (01)04006396098672(21)RU11120526160162049. Скобки и служебные символы (FNC1) добавляются только для наглядности/кодирования — при сканировании и в подписи они не мешают.

Количество кодов

Жёсткого ограничения на количество кодов за один раз нет. Для стабильной работы браузера и разумного размера ZIP-архива рекомендуется генерировать до нескольких сотен кодов (~500) за один раз.

Результат

После нажатия «Сгенерировать» ниже появится предпросмотр всех штрихкодов. Кнопка «Скачать ZIP (PNG)» упаковывает все сгенерированные изображения в один архив.
